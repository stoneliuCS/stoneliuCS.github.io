Learning about the compiler.

From my understanding a compiler is just a very, very, very sophisticated function. In essence it takes source code, and translates that into a semantically equivalent form in the target language.

;; The signature of a example compiler function in ISL.
compiler: String -> String
(define (compiler source)
  ...
)

Some popular compilers today include gcc, clang, javac, and go. So lets try writing my own, in C. The first stage is lexical analysis, essentially I need to write a program that can read in a source file and tokenize it. (I won't worry about preprocessing and other fancy stuff right now.)

// Reading from a source file in c.
printf("Compiling...\n");
const char* filePath = argv[1];
char* fileContents = readFile(filePath);

For now, lets try to translate this program written in an imaginary programming language called StoneScript (Creative am I right?) In my language, I prefer the arrow syntax to better communicate assignment statements.
